# docs-review-follow-up

Runtime compatibility: shared

## Purpose

Use this skill after `docs-review` or `docs-review-gate` finds documentation issues.

## Workflow

1. Read the docs review report, touched docs, relevant source evidence, and current git state.
2. Triage findings into fix now, defer, answer, withdrawn, or needs human decision.
3. Edit only documentation unless the user explicitly changes scope.
4. Self-check touched docs for accuracy, links, structure, style, and staleness.
5. Prepare an item-by-item docs review reply draft.

## Output

- Review report used
- Documentation changes
- Self-check results
- Finding disposition
- Reply draft
- Remaining documentation risk
