# ProjectSpec Alignment Policy

Use this policy when a repository has a project spec, implementation plan, DoD, or equivalent source of truth.

## Alignment Rules

- Read the relevant spec and plan before implementation.
- Map each task slice to a requirement, DoD item, or explicit user request.
- Do not silently expand product scope.
- If implementation reveals a source-of-truth conflict, stop before editing behavior unless the safe correction is obvious and low-risk.
- Update docs or status files when the repo policy requires it.

## Reporting

Reports should state:

- source-of-truth files read
- requirements covered
- requirements not covered
- assumptions
- open product or technical decisions
