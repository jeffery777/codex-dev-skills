# Naming

Skill names should be action-oriented, public, and stable.

Use:

- `planning`
- `implementation-slice`
- `project-delivery`
- `code-review-gate`
- `merge-readiness-gate`
- `desktop-project-orchestrator`

Avoid:

- private abbreviations
- runtime-internal names
- provider-specific labels
- names that imply hidden dependencies

Desktop-only skills may use the `desktop-` prefix because it communicates a real runtime boundary.
