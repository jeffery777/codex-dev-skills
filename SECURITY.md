# Security Policy

## Reporting

Please report security-sensitive issues privately through the repository owner or the platform security reporting channel if one is enabled.

## Scope

This repository contains workflow instructions and templates. Security review should focus on whether workflows could cause unsafe edits, credential exposure, destructive operations, data leakage, or misleading merge readiness claims.

## Handling Sensitive Material

Do not commit real credentials, private keys, local runtime state, logs, local databases, app state, or machine-specific config. Use placeholders only when an example is necessary, and keep the placeholder clearly non-real.
