# Merge Readiness Workflow

Runtime compatibility: shared

Use this workflow after implementation and review evidence exist.

1. Confirm base, head, repository identity, and changed files.
2. Read plans, DoD, prior reviews, verification output, and unresolved questions.
3. Check whether required review gates passed.
4. Verify that blockers were fixed with evidence.
5. Check docs, migration notes, release notes, or operational evidence when applicable.
6. Decide READY, BLOCKED, or NEEDS HUMAN DECISION.
7. Stop before merge, deploy, push, or publication unless the user explicitly authorized the exact action.
